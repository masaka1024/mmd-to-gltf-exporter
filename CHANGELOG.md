# Changelog

## v2.6.6
- スフィア専用フォールバックを追加。「銀」「髪止め」など、ベーステクスチャを持たずスフィアマップ（反射テクスチャ）だけで色や光沢を表現する金属系マテリアルが、diffuse/ambientともほぼ黒のために真っ黒になっていた問題を解消
  - ベーステクスチャがなく、計算した単色もほぼ黒の場合に限り、sphere_mode の設定に関わらずスフィアマップの色を採用する

## v2.6.5
- テクスチャ透過の方式を「ブレンド」から「クリップ（二値抜き）」に変更
  - 半透明の板が何枚も重なる髪・眉等の構造では、ブレンド方式は描画順に強く依存し、順序がずれるとレイヤーの奥側が透けて見える・裏返って見えるといった破綻を起こしていた。クリップはピクセル単位で透明/不透明を二値判定するため描画順に依存せず、この種の破綻が起きない
- Blender 4.2（EEVEE Next）の `surface_render_method` に `"OPAQUE"` が存在せず、不透明マテリアルの設定時に例外が出ていた問題を修正（色自体への実害はなかったが、透過設定の更新が阻害されていた）

## v2.6.4
- テクスチャのアルファチャンネルをPrincipled BSDFのAlphaソケットに接続する処理を追加
  - 眉毛のように「地肌の上に半透明で重ね描きされた」テクスチャが、変換後は不透明な黒として描画されてしまう問題を解消（マテリアルのアルファ値が1.0でも、テクスチャ自体に透過がある場合のみ反映）

## v2.6.3
- MMD式の色合成における diffuse 係数を 0.6 → 1.0 に修正
  - diffuse=白・ambient=黒という最も一般的な組み合わせでも、以前の式ではテクスチャ全体が一律40%暗くなってしまい、眉毛・瞳孔・輪郭線などの暗い部分が真っ黒に潰れる原因になっていた

## v2.6.2
- マテリアルの色合成式に ambient（環境色）を反映する仕組みを追加（`テクスチャ × clamp(diffuse + ambient)`）
  - diffuse が黒で ambient に色を持つマテリアルが真っ黒になる問題への対応として導入（このバージョン時点では別の副作用があり、v2.6.3 で調整）

## v2.6.1
- FBX エクスポートの「モディファイアを適用」を既定オフに変更。シェイプキー（= MMD モーフ：まばたき・口パク等）を保持したままエクスポートできるようにした
  - Blender はシェイプキーを持つメッシュにモディファイアを適用できないため、オンにするとモーフが失われる。FBX は座標軸で向きを処理でき、適用なしでもジオメトリが裏返らないことを確認（Unity で動作確認済み）

## v2.6.0
- FBX エクスポート機能を追加（Unity / Unreal Engine 向け）
  - `add_leaf_bones=False`、`mesh_smooth_type=FACE`、テクスチャ埋め込みなど、ゲームエンジン取り込みに適した既定値
  - 前処理（内部オブジェクト非表示・SDEF ミュート）は glTF と共通化
- アドオン名を「MMD to glTF Exporter」→「MMD Exporter」に変更（glTF / FBX 両対応のため）
- パネル Step 3 を「GLB / FBX」の出力選択に変更
